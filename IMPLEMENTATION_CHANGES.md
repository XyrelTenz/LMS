## Task 1: Add Copies Tracking to Books

### Overview
This task adds the ability to track how many copies of each book exist and prevents borrowing if copies <= 1.

### Part 1: Rust Backend Changes

**Status:** ✅ READY TO IMPLEMENT

#### 2. Database Migration (`rust/src/infrastructure/sqlite/mod.rs`)
**Status:** ✅ READY TO IMPLEMENT

Add migration to add `copies` column with DEFAULT 1:
```rust
// In init_db() function, add this migration:
let _ = conn.execute("ALTER TABLE books ADD COLUMN copies INTEGER NOT NULL DEFAULT 1", []);
```

This ensures:
- All existing books default to 1 copy
- No data loss or corruption
- Non-breaking change

#### 4. Service Layer Changes (`rust/src/application/book_service.rs`)
**Status:** 📝 NEEDS IMPLEMENTATION

Make the following changes:

**A. Update `add_book` function:**
```rust
pub fn add_book(
    title: String,
    author: String,
    publication_year: i32,
    isbn: String,
    genre: String,
    copies: Option<i32>,  // NEW
    image_url: Option<String>
) -> Result<(), String> {
    let conn = sqlite::init_db().map_err(|e| e.to_string())?;
    let id = Uuid::new_v4().to_string();
    let copies = copies.unwrap_or(1);  // Default to 1
    
    conn.execute(
        "INSERT INTO books (id, title, author, publication_year, isbn, genre, is_available, copies, image_url) VALUES (?1, ?2, ?3, ?4, ?5, ?6, 1, ?7, ?8)",
        params![id, title, author, publication_year, isbn, genre, copies, image_url],
    ).map_err(|e| e.to_string())?;
    Ok(())
}
```

**B. Update `get_all_books` function:**
- Add `copies` to the SELECT statement: `SELECT id, title, author, publication_year, isbn, genre, is_available, copies, image_url FROM books`
- Update the query_map to include: `copies: row.get(7)?,`

**C. Update `search_books` function:**
- Similar changes as get_all_books - add copies column to SELECT and query_map

**D. Update `update_book` function:**
- Include copies in the UPDATE statement (if you want to allow changing copies via update)
- Current version doesn't modify copies - adjust if needed

**E. Add borrowing logic (create new functions or modify existing borrowing functions):**

You'll need to find where borrowing/returning happens and:
- When a book is borrowed: `UPDATE books SET copies = copies - 1 WHERE id = ?`
- When a book is returned: `UPDATE books SET copies = copies + 1 WHERE id = ?`
- In the borrow endpoint, check: `IF copies <= 1, return error "Cannot borrow - last copy in use"`
